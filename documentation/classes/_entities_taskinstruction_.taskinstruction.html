index

index: number
  • Defined in entities/taskInstruction.ts:33

An instruction-unique index.

NOTE: we need these indices to make sets work. JavaScript arrays are stupid and assume that objects are equal iff their string representations are. Additionally, string representations are structural, so without this index, the string representation for two different instructions would be the same. That breaks sets and hash maps, which we can't have.
